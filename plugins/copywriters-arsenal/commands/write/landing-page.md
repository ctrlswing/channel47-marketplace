# Landing Page Writer

You are now in **Landing Page Writing Mode**.

## Input Required

1. **Product/Service**: What are we selling?
2. **Target Audience**: Who is this for?
3. **Awareness Level**: Where are they? (1-5)
4. **Primary Action**: What should they do? (Sign up, buy, book call, etc.)
5. **Key Differentiator**: What makes this unique?

## Landing Page Structure

### Above the Fold
```
HEADLINE
- Benefit-driven, specific
- Matched to awareness level

SUBHEADLINE
- Supports/expands headline
- Adds secondary benefit

HERO IMAGE/VISUAL
[Describe what should be shown]

PRIMARY CTA
- Clear action
- Benefit-oriented button text
```

### Problem Section
```
PROBLEM AGITATION
- Describe their current pain
- Use their language
- Build empathy
```

### Solution Section
```
SOLUTION INTRODUCTION
- Your unique mechanism
- How it works differently
- Bridge from problem to solution
```

### Benefits Section
```
KEY BENEFITS (3-5)
- Benefit 1: [Feature → Benefit → Outcome]
- Benefit 2: [Feature → Benefit → Outcome]
- Benefit 3: [Feature → Benefit → Outcome]
```

### Proof Section
```
SOCIAL PROOF
- Testimonials (specific)
- Logos/numbers
- Case study snippet
- Trust indicators
```

### How It Works
```
PROCESS (3 steps)
1. [Step 1]
2. [Step 2]
3. [Step 3]
```

### Offer Section
```
WHAT YOU GET
- Main offer
- Bonuses (if applicable)
- Value stack

PRICING
- Price with context
- Payment options

GUARANTEE
- Risk reversal
- Confidence statement
```

### Final CTA Section
```
CLOSING STATEMENT
- Restate transformation
- Address final objection

CTA BUTTON
- Action-oriented text
```

### FAQ Section
```
COMMON QUESTIONS
- Objection 1 → Answer
- Objection 2 → Answer
- Objection 3 → Answer
```

## Framework Selection

Based on the product and audience, I'll apply:

| Awareness Level | Framework Approach |
|----------------|-------------------|
| Most Aware | Ogilvy (proof-heavy, offer-first) |
| Product Aware | Carlton (differentiation, voice) |
| Solution Aware | Schwartz (mechanism focus) |
| Problem Aware | Dickens (vivid problem/solution) |
| Unaware | Homer (story-driven) |

## Output Format

Provide full landing page copy with:

1. **Section-by-section copy**
2. **Headline alternatives** (3 options)
3. **CTA button text options** (3 options)
4. **Framework notes** (why each section works)

## Quality Checks

- [ ] Headline passes the 3-second test
- [ ] One primary CTA (repeated appropriately)
- [ ] Proof is specific and credible
- [ ] Benefits > Features throughout
- [ ] Mobile-friendly length for key sections
- [ ] Urgency/scarcity if appropriate
- [ ] Guarantee clearly stated

Now write the landing page based on the user's input.
